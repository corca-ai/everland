# 문서 작성 가이드

이 문서는 프로젝트 문서를 작성하고 유지하는 방법을 정의합니다.

## 목표

사람과 에이전트 모두가 빠르게 훑어보고 신뢰할 수 있는 문서를 유지합니다.

## 원칙

- 문서를 코드처럼 다룹니다.
- 의도를 명확하고 간결하게 유지합니다.
- 길고 뒤섞인 문서보다 짧고 집중된 문서를 선호합니다.
- 관련 문서를 링크하되, 탐색 경로는 짧게 유지합니다.
- 자동 생성된 문서는 버전 관리하지 않습니다.

## 구조

- `README.md`는 최소한으로 유지하고 `AGENTS.md`를 가리킵니다.
- `AGENTS.md`는 최소한으로 유지하고 상세 문서를 링크합니다.
- 주제별 문서(예: `slack-setup.md`, `game-dev.md`)는 해당 주제에 집중하고 상위 문서에서 링크합니다.

## 작성 규칙

- 현재 아키텍처와 일관된 용어를 사용합니다.
- 더 이상 사용하지 않는 아키텍처 용어는 즉시 제거합니다.
- 현재 런타임 동작과 일치하는 예시를 사용합니다.

참고: `CLAUDE.md`는 `AGENTS.md`의 심볼릭 링크입니다.
